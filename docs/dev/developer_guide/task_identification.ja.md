# タスクディレクトリの特定

コマンドが `docs/tasks/` 配下の対象タスクディレクトリを特定する必要がある場合、以下の優先規則を順番に適用する。

1. **引数**: `$ARGUMENTS` が空でない場合、タスク識別子（例：`0020` または `0020_tlsrpt`）として解釈し、`docs/tasks/` 配下の一致するディレクトリに解決する。一致するディレクトリがなければエラーで停止する。
2. **開いているファイル**: `docs/tasks/<task>/` 配下のファイルが IDE で開かれている場合、そのタスクディレクトリを使用する。
3. **曖昧**: いずれの規則も適用できない場合は停止する。候補となるタスクディレクトリを一覧表示し、ユーザーに引数として明示的なタスク識別子を指定して再実行するよう求める（例：`/mkarch 0020`）。
