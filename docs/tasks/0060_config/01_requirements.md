## 1. 背景と目的

### 1.1 背景

tlsrpt-digest は IMAP 接続情報、通知先、ポーリング間隔など複数の設定項目を持つ。
これらを TOML 形式の設定ファイルから読み込む機能を実装する。

### 1.2 目的

1. **主目的**: TOML 設定ファイルを読み込み、アプリケーション設定を構造体として提供する
2. **副次的目的**: 設定値のバリデーションにより早期に設定ミスを検出する

---

## 2. スコープ

### 対象範囲（In Scope）

- TOML 設定ファイルの読み込みと Go 構造体へのデコード
- 必須設定項目の存在チェック
- 設定値のバリデーション（型・範囲・形式）
- コマンドライン引数による設定ファイルパスの指定

### 対象外（Out of Scope）

- 設定ファイルのホットリロード（アプリ再起動で反映）
- 環境変数による設定上書き（将来の拡張）
- 設定ファイルの暗号化

### 影響を受けるコンポーネント

- **直接変更**: `internal/config/`（新規作成）または `cmd/tlsrpt-digest/` 内
- **間接的影響**: `cmd/tlsrpt-digest/`（設定の利用側）

---

## 3. 機能要件

### F-001: 設定ファイルの読み込み

指定されたパスの TOML ファイルを読み込み、`Config` 構造体にデコードする。

**受け入れ条件（Acceptance Criteria）**:

1. 有効な TOML ファイルを指定した場合、すべての設定値を正しく読み込む
2. 指定ファイルが存在しない場合、エラーを返す
3. TOML の文法エラーがある場合、エラーを返す
4. 未知のキーが含まれる場合、エラーまたは警告を出力する（strict モード）

### F-002: 設定値のバリデーション

読み込んだ設定値が有効であることを検証する。

**受け入れ条件（Acceptance Criteria）**:

1. IMAP ホスト名が空の場合はエラーを返す
2. IMAP ポート番号が 1〜65535 の範囲外の場合はエラーを返す
3. IMAP ユーザ名またはパスワードが空の場合はエラーを返す
4. 通知設定（Slack Webhook URL またはメール設定）が一つも設定されていない場合はエラーを返す
5. ポーリング間隔が 0 以下の場合はエラーを返す

### F-003: デフォルト値の適用

省略可能な設定項目にはデフォルト値を適用する。

**受け入れ条件（Acceptance Criteria）**:

1. ポーリング間隔が未設定の場合、デフォルト値（例：1 時間）が適用される
2. IMAP メールボックス名が未設定の場合、デフォルト値（`"INBOX"`）が適用される
3. SQLite パスが未設定の場合、デフォルト値（例：`"./tlsrpt.db"`）が適用される

---

## 4. 非機能要件

### セキュリティ

- パスワードや Webhook URL が設定ファイルのパースエラーメッセージに含まれないこと

### 保守性

- `Config` 構造体は各パッケージ（imap、notify、store）の設定をネストした形で持つ

---

## 5. 制約

- 使用言語は Go とする（Go 1.23 以上）
- TOML ライブラリには `BurntSushi/toml` を使用する
- テストには `stretchr/testify` を使用する

---

## 6. テスト方針

### 単体テスト

- 有効な設定ファイルの読み込みテスト
- 必須項目欠如時のバリデーションエラーテスト
- デフォルト値適用のテスト
- 不正 TOML（文法エラー）のエラーテスト
